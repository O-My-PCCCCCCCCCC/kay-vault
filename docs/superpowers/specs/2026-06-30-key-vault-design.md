# 凯伊密码管家 (Key Vault) — 设计规格

> 基于 Blue Archive 角色 **Key (凯伊/柯伊)** 主题的密码管理器 + 密码生成器桌面应用
> 日期：2026-06-30

---

## 1. 项目概述

### 1.1 目标
一款运行在 U 盘上的跨平台（Windows + Linux）密码管理器，以密码管理为主导功能，密码生成为辅助工具。支持加密本地存储、U 盘密钥认证、加密备份与还原。

### 1.2 技术选型
| 层 | 技术 | 理由 |
|----|------|------|
| 前端 | **Vue 3 + TypeScript** | 生态成熟，UI 精美度上限高 |
| UI 组件 | **Naive UI** | 组件丰富，暗色主题支持好，适合桌面风格 |
| 构建打包 | **Tauri 2.0** | 单文件 ~5-10MB，跨 Windows + Linux，放 U 盘即开即用 |
| 后端 | **Rust** | 密码学生态成熟（aes-gcm, argon2），性能强，安全硬核 |
| 加密标准 | AES-256-GCM + Argon2id | 业界标准对称加密 |

### 1.3 角色主题
- **角色：** Blue Archive — Key (凯伊/柯伊)
- **配色方案：**
  - 主色：深空黑 `#0a0e1a` → 白银 `#e8e8e8`
  - 强调色：赤红 `#dc2626`（红瞳意象）
  - 辅助色：冰蓝 `#7ec8e3`（千年学园制服）
  - 渐变色：红瞳渐变 `#dc2626 → #991b1b`
- **氛围：** 简洁锐利、高级感、玻璃拟态（glassmorphism）
- **彩蛋元素：** 钥匙图标 logo、空状态/锁屏文案彩蛋

## 2. 页面结构

| 页面 | 类型 | 功能 |
|------|------|------|
| **我的密码** | 🏠 主页 | 卡片列表展示所有密码条目，搜索、分类筛选、CRUD |
| **设置** | ⚙️ 次页 | 主密码修改、自动锁定、认证设备管理、备份/还原 |
| **密码生成器** | 🔐 弹出面板 | 嵌入在新建/编辑表单底部的工具，非独立页面 |

### 2.1 导航
- 左侧精简侧边栏（icon + 文字）
- 主页（我的密码）为启动默认页
- 设置入口放在侧栏底部

### 2.2 我的密码（主页）
- 顶部：搜索栏 + 分类筛选下拉
- 主体：密码卡片网格，每张卡片显示：名称、用户名、分类标签、更新时间
- 悬停卡片：显示「复制账号」「复制密码」「编辑」「删除」快捷操作按钮
- 右下角浮动「+」按钮 → 弹出新建表单

### 2.3 新建/编辑表单
- 字段：名称、URL、用户名、密码、备注、分类
- 密码字段右侧：**🎲 生成密码** 按钮 → 弹出密码生成面板
- 密码生成面板选项：长度、包含大写/小写/数字/符号、排除相似字符、一键填入

### 2.4 设置页面
- 安全设置：修改主密码、自动锁定时间
- 设备认证：已认证设备列表（显示设备名 + 系统类型）、添加/删除认证
- 备份与还原：备份到 U 盘、从 U 盘还原

## 3. 数据模型

### 3.1 密码条目 (VaultEntry)

```typescript
interface VaultEntry {
  id: string;            // UUID v4
  name: string;          // 条目名称（如 "GitHub"）
  url: string;           // 关联网站/应用
  username: string;      // 账号
  password: string;      // 密码（AES-GCM 加密存储）
  notes: string;         // 备注
  category: string;      // 分类标签
  created_at: string;    // ISO 8601
  updated_at: string;    // ISO 8601
}
```

### 3.2 分类

预设分类（可自定义）：
- 社交账号
- 开发工具
- 金融支付
- 邮箱
- 娱乐游戏
- 其他

### 3.3 本地配置 (Config)

```typescript
interface AppConfig {
  vault_path: string;          // 加密库路径
  auto_lock_minutes: number;   // 自动锁定时间（默认 5）

  theme: 'dark' | 'light';    // 主题模式
  last_unlock: string;         // 上次解锁时间 ISO 8601
}
```

## 4. 加密体系

### 4.1 加密流程

```
主密码（用户记忆）
    │
    ▼
Argon2id 密钥派生（慢哈希，高迭代）
    │
    ▼
AES-256-GCM 对称加密密钥（256-bit）
    │
    ▼
vault.enc — 加密后的密码库文件
（JSON 序列化 → AES-GCM 加密 → 写入磁盘）
```

- **Argon2id 参数：** 内存 64MB，迭代 3 次，并行 4 线程
- **AES-GCM：** 每次加密生成随机 nonce（12 字节），附带认证标签
- **vault.enc 文件结构：** `[nonce(12B)][ciphertext][auth_tag(16B)]`
- 明文不在磁盘停留，解密后驻内存，锁定/退出时擦除

### 4.2 文件存储位置

| 文件 | 路径 | 说明 |
|------|------|------|
| 加密密码库 | `~/.key-vault/vault.enc` | AES-GCM 加密的密码库 |
| 配置文件 | `~/.key-vault/config.json` | 明文存储，不含敏感信息 |

## 5. 认证与备份体系

### 5.1 U 盘目录结构

```
<U盘>/.key-vault/
├── keys/              # 设备密钥目录
│   ├── PC-A.key       # 电脑 A 的认证密钥
│   └── PC-B.key       # 电脑 B 的认证密钥
└── backups/           # 备份目录
    └── vault-20260630.enc   # 加密备份文件
```

### 5.2 设备认证流程

```
首次使用：
  1. 插入 U 盘
  2. 程序检测到无本地密钥
  3. 用户点击「添加设备认证」
  4. 程序生成设备唯一密钥文件 → 写入 U 盘 keys/
  5. 完成认证

日常使用（备份/还原）：
  1. 插入 U 盘
  2. 程序遍历 U 盘 keys/ 目录
  3. 查找是否有本机标识对应的密钥文件
  4. 有 → 认证通过，允许备份/还原操作
  5. 无 → 认证失败，提示「该设备未认证」

删除认证：
  1. 在设置页 → 设备认证管理 → 选择设备 → 点「删除」
  2. 弹出二次确认对话框
  3. 确认后删除 U 盘上对应的密钥文件
```

### 5.3 本机标识方案
- 使用 `hostname + 机器指纹` 生成唯一设备 ID
- 密钥文件内容：加密后的设备 ID + 随机 token
- 不依赖网络、不依赖硬件绑定（U 盘密钥即认证）

### 5.4 备份/还原流程

```
备份（本机 → U 盘）：
  1. 插入 U 盘
  2. 认证通过
  3. 用户点击「备份」
  4. 程序从 vault.enc 读取当前加密库
  5. 复制到 U 盘 backups/vault-<YYYYMMDD>.enc
  6. 提示「备份完成」

还原（U 盘 → 本机）：
  1. 插入 U 盘
  2. 认证通过
  3. 用户点击「还原」
  4. 弹出二次确认：「还原将覆盖当前所有密码数据，确定继续？」
  5. 确认 → 从 U 盘备份文件复制到本机 vault.enc
  6. 提示「还原完成 → 请重新解锁」
```

## 7. 密码生成器（辅助功能）

### 7.1 配置选项

| 选项 | 默认值 | 范围 |
|------|--------|------|
| 密码长度 | 16 | 4 ~ 128 |
| 包含大写字母 | ✓ | — |
| 包含小写字母 | ✓ | — |
| 包含数字 | ✓ | — |
| 包含符号 | ✗ | `!@#$%^&*()_+-=[]{}|;:,.<>?` |
| 排除相似字符 | ✗ | 排除 `0OIl1` 等易混淆字符 |

### 7.2 安全性
- 使用 `window.crypto.getRandomValues()` 生成随机数
- 生成结果不存储不记录，填入选框即丢弃

## 8. 安全注意事项

| 项目 | 措施 |
|------|------|
| 加密算法 | AES-256-GCM，带认证标签防篡改 |
| 密钥派生 | Argon2id，抗 GPU/ASIC 暴力破解 |
| 内存安全 | Rust 内存安全，无 GC |
| 密码明文 | 只在解锁状态下驻内存，锁定/退出擦除 |
| U 盘认证 | 无网络依赖，密钥物理隔离 |
| 二次确认 | 删除认证、还原备份等危险操作需确认 |
